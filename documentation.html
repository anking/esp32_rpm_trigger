🎯 Project Overview

This project implements a sophisticated NOS (Nitrous Oxide System) controller using an ESP32-S3 microcontroller that interfaces with your vehicle's OBD-II port via Bluetooth. The system monitors real-time engine parameters and provides intelligent control over nitrous injection systems.

🔗 Wireless OBD-II Communication

Connects to ELM327 Bluetooth adapters for real-time vehicle data monitoring

⚡ Real-time Engine Monitoring

Monitors RPM, throttle position, and vehicle speed with adaptive polling

🎮 Intelligent NOS Control

Manual and automatic nitrous injection modes with safety parameters

💡 Visual Status Indicators

RGB LED and relay-controlled external LED indicators for system status

🌟 Key Features

Core Functionality

  • Bluetooth OBD-II Integration: Wireless connection to ELM327 adapters
  • Multi-PID Data Collection: Efficient single-request polling for RPM, throttle, and speed
  • Adaptive Communication: Dynamic delay adjustment based on ECU response times
  • Robust Error Recovery: Automatic reconnection and error handling

NOS Control System

  • Manual Mode: Button-activated nitrous injection
  • Automatic Mode: RPM and throttle-based automatic activation
  • Safety Thresholds: Configurable RPM and throttle position limits
  • Relay-based Outputs: High-current relay control for nitrous solenoids

Status Indication

  • Bluetooth Status: Connection state indication
  • ECU Status: Vehicle communication status
  • NOS Status: System readiness and mode indication

📥 Installation & Setup

1. Software Installation

# Install PlatformIO Core pip install platformio # Clone the repository git clone <repository-url> cd BT-NOS-Controller # Build and upload firmware pio run -t upload -t monitor

2. Hardware Setup

  1. Connect ESP32-S3: Mount the ESP32-S3 on the relay board
  2. Connect External Components: Wire NOS solenoids and status LEDs to relays
  3. Install ELM327: Plug ELM327 adapter into vehicle OBD-II port
  4. Power Connection: Connect 12V power supply to relay board

3. First Boot Configuration

  1. Power on the system
  2. Monitor serial output for initialization messages
  3. Pair with ELM327 adapter (auto-discovery enabled)
  4. Start vehicle engine for ECU connection test
  5. Verify status LED operation
✅ Success Indicators: Blue LED for Bluetooth, Green LED for ECU connection, and real-time RPM/throttle/speed data in serial monitor.

🎮 Operation Guide

System Startup Sequence

  1. Power On: RGB LED shows initialization sequence
  2. Bluetooth Scan: Blue LED blinks while searching for ELM327
  3. ELM327 Connection: Blue LED solid when connected
  4. ECU Communication: Green LED blinks during ECU initialization
  5. System Ready: Green LED solid, NOS system operational

Operating Modes

Manual Mode (Default)

  • NOS activation requires manual button press
  • Red LED solid when NOS conditions are met
  • Press BOOT button to activate NOS injection
  • Safety thresholds still apply

Automatic Mode

  • Toggle mode by holding BOOT button for 3 seconds
  • Red LED slow blink indicates auto mode active
  • NOS activates automatically when conditions are met:
    • RPM > 2000 (configurable)
    • Throttle > 80% (configurable)
    • Vehicle speed > 30 km/h (configurable)

Status Monitoring

🔧 Troubleshooting

Common Issues

Bluetooth Connection Problems

  • Blue LED not lighting: Check ELM327 adapter power and pairing
  • Connection drops: Verify ELM327 adapter quality and distance
  • Slow response: Check for interference from other Bluetooth devices

ECU Communication Issues

  • Green LED not lighting: Ensure engine is running (not just ignition ON)
  • "NO DATA" errors: Try cycling ignition OFF/ON and restart engine
  • Intermittent connection: Check OBD-II connector for loose connections

Honda Civic 2018 Specific

Honda Civic 2018 Troubleshooting:
  • Engine MUST be running (not just ignition ON)
  • Press accelerator or turn on A/C to wake up PCM
  • Hood should be open (Honda security feature)
  • Wait 15s after engine start (PCM initialization)
  • Try cycling ignition: OFF (10s) → ON → Start Engine

NOS System Issues

  • Red LED not activating: Check safety threshold settings
  • Manual mode not working: Verify button wiring and relay connections
  • Auto mode not engaging: Check RPM/throttle/speed thresholds

Debug Information

Enable debug logging by setting ENABLE_DEBUG_LOGGING = 1 in include/logging_config.h to see detailed communication logs.

⚠️ Safety Information

CRITICAL SAFETY WARNINGS

  • High Pressure Systems: NOS systems operate at extremely high pressures. Ensure all connections are rated for the pressure.
  • Electrical Safety: Use properly rated relays and fuses for all high-current connections.
  • Vehicle Integration: Improper installation can damage vehicle electronics. Consult professionals if unsure.
  • Legal Compliance: Check local regulations regarding NOS systems and vehicle modifications.
  • Emergency Shutdown: Always include manual emergency cutoff switches for safety.

Safety Features

  • Threshold Monitoring: System monitors RPM, throttle, and speed limits
  • Fail-Safe Operation: System defaults to safe state on errors
  • Connection Monitoring: Automatic shutdown if communication is lost
  • Relay Isolation: High-current loads isolated from microcontroller

📊 Technical Specifications

Microcontroller Specifications

Parameter Value Notes
Microcontroller ESP32-S3 Dual-core Xtensa LX7, 240 MHz
Flash Memory 8 MB For firmware and data storage
RAM 512 KB SRAM System and application memory
Bluetooth Bluetooth LE 5.0 Low energy for extended operation
Operating Voltage 3.3V Regulated from 5V USB or external supply

Communication Specifications

  • OBD-II Protocol: ISO 15765-4 (CAN), ISO 14230-4 (KWP2000)
  • Bluetooth Range: ~10 meters (Class 2)
  • Data Update Rate: 5-10 Hz (adaptive based on ECU response)
  • Response Timeout: 1000ms with exponential backoff

Relay Specifications

  • Contact Rating: 10A @ 250VAC, 10A @ 30VDC
  • Coil Voltage: 3.3V (driven directly from GPIO)
  • Switching Time: ~10ms typical
  • Contact Configuration: SPDT (Single Pole, Double Throw)

Software Architecture

  • RTOS: FreeRTOS with multiple tasks
  • Task Structure:
    • Bluetooth communication task
    • OBD data polling task
    • GPIO control task
    • LED indicator task
  • Memory Management: Static allocation with overflow protection
  • Error Handling: Comprehensive error recovery and logging
